# Swagger API 구현 스킬

사용자가 API 연동을 요청하면 아래 단계를 반드시 따를 것.

## 절차

1. **스펙 확인**: 사용자가 제공한 Swagger/API 스펙 또는 백엔드 DTO/Entity 파일을 먼저 읽는다
2. **필드 목록 확인**: 필요한 필드와 정확한 타입/enum 값을 나열하고 사용자에게 확인받는다
3. **불필요한 필드 금지**: 스펙에 명시된 필드만 구현한다 — 추가 필드 절대 금지
4. **enum 값 정확히 일치**: emotion_types, weather types, UserType 등 enum 값은 백엔드 정의와 100% 동일하게 사용
5. **기존 패턴 확인**: `/services`, `/api`, `/hooks` 디렉토리에서 기존 API 호출 패턴을 먼저 확인하고 동일한 패턴으로 구현
6. **기존 컴포넌트 사용**: UI 구현 시 프로젝트의 커스텀 컴포넌트를 사용 (raw HTML 요소 사용 금지)
7. **페이로드 검증**: 구현 후 요청/응답 페이로드가 스펙과 일치하는지 최종 확인

## 주의사항
- `any` 타입 사용 금지
- 선택적 필드는 optional로 정확히 표시
- 날짜 형식은 백엔드와 동일하게 (ISO 8601 등)
